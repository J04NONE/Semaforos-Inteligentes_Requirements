# 🚦 TrafficUribe-Requirements
Control de versiones para la gestión de requisitos del sistema de semaforización inteligente.

# 🏢 Sistema de Gestión de Semáforos: Localidad Rafael Uribe Uribe

Software de misión crítica diseñado para la automatización, sincronización y monitoreo del flujo vehicular en la localidad 18 de Bogotá, bajo estándares de Sistemas Inteligentes de Transporte (SIT).

## 🎯 Objetivo del Proyecto
Centralizar y optimizar la gestión de requisitos para el software de control de tráfico, asegurando la interoperabilidad entre sensores de flujo, controladores locales y el Centro de Gestión de Tránsito (CGT).

## 📁 Estructura del Proyecto
* [📖 Documentación IEEE 830](./docs/srs_ieee830.md) - Especificación formal de requisitos (RF y RNF).
* [📝 Historias de Usuario](./backlog/user_stories_traffic.md) - Requerimientos desde la perspectiva del operador y el ciudadano.
* [📊 Matriz de Trazabilidad](./validacion/matriz_trazabilidad.md) - Vinculación de sensores e inputs con funcionalidades del sistema.
* [🚨 Protocolos de Fallo](./docs/fail_safe_protocols.md) - Definición de estados críticos (Modo intermitente).
